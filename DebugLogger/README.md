## Описание

`DebugLogger` — это легкая и удобная библиотека является "оберткой" `Serial.print` для отладочного вывода в последовательный порт платформы Arduino. Она предоставляет интуитивно понятный синтаксис, схожий с `std::cout`, с помощью перегрузки оператора `<<`.

Основные особенности:

- **Простой синтаксис:** Используйте `Debug << "message" << myVariable << endl;`.
- **Поддержка различных типов данных:** `const char*`, `char`, `int`, `unsigned int`, `long`, `unsigned long`, `double`, `String`.
- **Манипуляторы форматирования:** Удобный вывод чисел в шестнадцатеричном (`hex`), десятичном (`dec`), восьмеричном (`oct`) и двоичном (`bin`) форматах.
- **Отключение вывода:** Весь отладочный код может быть полностью отключен на этапе компиляции с помощью макроса `#define DEBUG 0`, что минимизирует размер прошивки.
- **Совместимость:** Поддерживает экосистему PlatformIO для платформ Arduino.

## Установка

Библиотека разработана для использования в PlatformIO.

1. **Создайте новый проект PlatformIO.**
2. **Откройте файл `platformio.ini`** и добавьте следующую строку в секцию `[env:]` для вашего проекта, чтобы указать путь к библиотеке:

    ```ini
    lib_deps =
        // Если библиотека находится в вашей локальной файловой системе
        // file:///path/to/DebugLogger
        // Если вы загрузите её на GitHub, вы можете использовать
        // [https://github.com/ThinkerSan/DebugLogger.git](https://github.com/ThinkerSan/DebugLogger.git)
    ```

    Или, если я опубликую библиотеку в PlatformIO Registry, просто укажите её имя.

## Использование

### 1. Подключение и инициализация

Включите `DebugLogger` в ваш код, определите макрос `DEBUG` и инициализируйте порт в функции `setup()`.

```cpp
#include <Arduino.h>
#include "DebugLogger.h"

// Включите этот макрос, чтобы активировать отладочный вывод
#define DEBUG 1

void setup() {
  // Инициализация последовательного порта со скоростью 9600 бод
  Debug.begin(9600);
}

void loop() {
  // Ваш код
}
```

### 2. Примеры вывода

#### Простой вывод текста и чисел

```cpp
void loop() {
  int value = 42;
  double pi = 3.14;
  
  Debug << "The answer is " << value << endl;
  Debug << "The value of PI is " << pi << endl;
  
  delay(1000);
}
```

#### Использование манипуляторов

Манипуляторы `hex`, `oct` и `bin` влияют только на следующее за ними числовое значение. После вывода формат сбрасывается к десятичному (`dec`) по умолчанию.

| Манипулятор | Назначение                 |
|-------------|----------------------------|
| `endl`      | Перевод строки             |
| `hex`       | Шестнадцатеричный формат   |
| `dec`       | Десятичный формат (по ум.) |
| `oct`       | Восьмеричный формат        |
| `bin`       | Двоичный формат            |


```cpp
void loop() {
  int myNumber = 255;
  
  Debug << "Decimal: " << myNumber << endl; // 255
  Debug << "Hexadecimal: " << hex << myNumber << endl; // ff
  Debug << "Octal: " << oct << myNumber << endl; // 377
  Debug << "Binary: " << bin << myNumber << endl; // 11111111

  // Пример цепочки вызовов
  Debug << "Value: " << dec << myNumber << " (hex: " << hex << myNumber << ")" << endl;
  
  delay(5000);
}
```

### Отключение отладочного вывода

Чтобы полностью отключить весь отладочный код, просто измените значение макроса `DEBUG` на `0` или не определяйте его вовсе. Весь код внутри `#if DEBUG` будет проигнорирован компилятором, что не скажется на размере и производительности вашей финальной прошивки.

```cpp
// Отключит весь вывод. Компилятор не будет включать этот код в прошивку.
#define DEBUG 0
```

### Хранение строк во FLASH, вместо RAM

Для хранения строк во Flash, а не в ОЗУ можно использовать макрос `F` :

```cpp
Debug << F("Iteration #") << counter << F(": Sensor value = ") << sensorValue << endl;
```
